{{ _("Compilers' Version and Parameters") }}

{{ name }} uses HydroJudge for judging. See the status page for compiler versions and flags.

{{ _('Limitations') }}

Time usage is measured by CPU time. Memory usage is measured by the process memory footprint.

Default memory limit is 256MiB unless a problem states a different limit.

{{ _('IO') }}

Unless stated otherwise, use standard input and standard output (STD I/O).

{{ _('Judge Status') }}

  • Waiting: queued and waiting for judge worker.
  • Fetched: picked by judge worker, preparing.
  • Compiling: source code is compiling.
  • Judging: running testcases.
  • Accepted: output is correct.
  • Wrong Answer: output differs from expected answer.
  • Time Limit Exceeded: runtime exceeded limit.
  • Memory Limit Exceeded: memory usage exceeded limit.
  • Runtime Error: runtime failure (e.g. crash or invalid operation).
  • Compile Error: compilation failed.
  • System Error: internal judge/platform error.
  • Canceled: judging was canceled.
  • Unknown Error: unknown failure reason.
  • Ignored: record was ignored.

{{ _('Compile Error') }}

Common causes:

  • Selected language does not match your code.
  • Java main class is not named Main.
  • General syntax or build issues.

{{ _('Difficulty Algorithm') }}

Problem difficulty is estimated from submissions, acceptance behavior, and evaluation history.

  • New problems may have unstable difficulty values.
  • Difficulty becomes more stable with more submissions.
  • Algorithmic estimation may still be imperfect.
